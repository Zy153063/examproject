我的系统一共有以上几种功能；
主要的操作的实现及理解为：
分为两个队列，混合检测队列和单管检测队列，可以输入人员代码进行分别排队，状态即为排队中，可以分别进行最前排人员检测，状态即变为等待上传结果，此时的人员会获得一个试管号，标记进行的检测试管，
登记结果根据试管号进行结果登记，若果有阳性确诊的话，会根据楼号和排队情况进行密接和次密接的设置。
首先要声明以下几个方面：
①题目中说确诊前十人和后面的人设置为密接，我在此理解为后面一人。
②题目中说读取文件时，要准备混合检测和单管检测的人数，我在实际操作和对实际情况的理解是，混合检测不能规定人数，或者说只能规定十的倍数，所以我在文件读取时输入的 混合检测选择是读取次数。
③由于我的系统设置为一个人只能带一种状态，所以我规定的状态优先级为确诊>密接>次密接 。
④由于题目说明了，检测人员不能超过排队人员，所以该系统没有在空队伍的基础上进行检测的功能。
操作建议：
1、首先（a）进行文件读取入队
2、选择（b）然后此时自行选择可以进行手动入队
3、选择（g）进行手动核酸检测
4、选择（d）查看排队情况
5、选择（e）分类展示人员中的
6、选择（d）登记人员状态 注意只有单管才有确诊状态
7、选择（e）分类查询状态，查询阳性，密接，次密接
8、选择（f） 查询个人信息的功能
详细操作和介绍看报告中的题目二。
